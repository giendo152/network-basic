# Проблема1: 

# Дано:                             
| <!-- -->                                    | <!-- -->              |
| ------------------------------------------- | --------------------- |
| IP-адрес узла:                             | 192.168.200.139        |
| Исходная маска подсети:                    | 255.255.255.0          |
| Новая маска подсети:                       | 255.255.255.224        |

# Найти: 
| <!-- -->                                    | <!-- -->              | 
| ------------------------------------------- | --------------------- | 
| Количество бит подсети                      |        3              |
| Количество созданных подсетей               |        8              |
| Количество бит узлов в подсети              |        5              |
| Количество узлов в подсети	                 |        30             |
| Сетевой адрес этой подсети                  | 192.168.200.128       |
| IPv4-адрес первого узла в этой подсети	     | 192.168.200.129       |
| IPv4-адрес последнего узла в этой подсети	  | 192.168.200.158       |
| Широковещательный IPv4-адрес в этой подсети | 192.168.200.159        |

# Решение:

Исходная маска подсети имела вид 255.255.255.0 или /24. Новая маска подсети — 255.255.255.224 или /27. 
Полученная разница составляет 3 бита. Поскольку 3 бита были заимствованы, мы можем определить, 
что были созданы 8 подсетей, так как 2^3 = 8.
В новой маске, равной 255.255.255.224 или /27, остается 5 бит для узлов. Если для узлов осталось
5 бит, воспользуемся следующей формулой: 2^5 = 32 – 2 = 30 узла для каждой подсети.
Побитовая операция И поможет определить подсеть для этой задачи, в результате чего мы получим
сеть 192.168.200.128.
